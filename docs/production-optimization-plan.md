# 生产环境优化计划

本文档记录平台面向约 1000 名学生和教师使用时的优化路线。优先级按上线风险排序，先保证数据安全、登录稳定和提交可靠，再逐步提升性能和体验。

详细评估见 [正式使用前生产就绪评估](production-readiness-review.md)。

## P0 已落地

- 自动备份：服务启动时自动备份一次，之后默认每 24 小时备份一次，备份目录为 `backups/`。
- 手动备份：管理员可在“管理后台 - 系统设置 - 数据备份”点击“一键备份”，也可执行 `npm run backup` 立即生成一份备份。
- 备份保留：默认保留最近 14 份，可通过 `BACKUP_RETENTION` 调整。
- 试卷存储：试卷运行数据写入 SQLite 的 `app_state.papers`，同时继续同步 `data/papers.json` 作为人工检查和恢复镜像。
- 原子写入：JSON 文件写入改为临时文件加重命名，降低写一半导致文件损坏的风险。
- 登录会话：登录 Cookie 增加有效期，默认 7 天，可通过 `SESSION_MAX_AGE_DAYS` 调整；HTTPS 反向代理下自动使用 `Secure` Cookie。
- 登录保护：同一 IP 和用户名在短时间内失败次数过多会临时限制登录，降低暴力尝试风险。
- 提交记录容量：答题记录默认保留 100000 条，可通过 `MAX_ATTEMPTS` 调整，避免 1000 人规模下很快覆盖成绩。
- 操作日志容量：教师和管理员操作日志默认保留 10000 条，可通过 `MAX_AUDIT_LOGS` 调整。
- 权限兜底：只有学生账号可以通过邀请码加入班级；教师和管理员仍通过管理台维护班级。
- 请求错误处理：JSON 格式错误、请求体过大、上传过大返回明确的 400 错误，避免被当成服务器故障。
- 提交记录结构化：新增 SQLite `attempts` 表和索引，学生提交写入独立表；学习中心、班级学情、用户管理和教师概览改为查询独立表；旧运行状态中的 `attempts` 会自动迁移，备份和恢复也包含独立提交记录表。

## P0 后续专项

这些属于较大改造，建议单独安排维护窗口完成。

- 提交记录分页完善：提交写入和主要统计已拆到独立表，后续还需给学习中心、班级学情、最近答题和历史记录增加分页参数，避免单次返回过多提交记录。
- 核心运行数据拆表：将 `users`、`sessions`、`classes`、`enrollments`、`assignments`、`auditLogs`、`paperVisibility` 拆成 SQLite 表，降低并发覆盖风险。
- 试卷结构化入库和编辑冲突检测：先增加保存版本冲突提示，再逐步将试卷、题目、选项、答案拆成 SQLite 表，便于分页、搜索、并发编辑和审计。
- 恢复演练和离线备份：定期在测试目录或新服务器上用 `backups/` 恢复一次，并把备份同步到服务器外部存储。

## P1 强烈建议

- Nginx 反向代理：负责域名、HTTPS、静态文件缓存、上传大小限制和访问日志。
- HTTPS：使用 Let’s Encrypt 为正式域名配置证书，保证登录 Cookie 和浏览器安全策略稳定。
- PM2 运行策略：在当前数据结构完全表结构化之前，建议先使用单实例 PM2，避免多进程同时写运行状态造成覆盖；结构化入库后再评估 cluster。
- 静态资源版本：为前端 JS/CSS 增加版本号或构建哈希，减少浏览器缓存旧文件导致的功能不一致。
- 后台分页和后端筛选：班级学情、最近答题、学生列表、用户列表、试卷列表都应支持分页和筛选。
- 日志轮转：配置 `pm2-logrotate`，避免日志文件长期增长占满磁盘。
- 服务器监控：至少监控 CPU、内存、磁盘、PM2 进程状态和接口 500 错误数量。
- 默认账号处理：正式上线前删除或停用演示账号，管理员使用强密码。

## P2 逐步完善

- 自动保存答题：学生答题过程中周期性保存草稿，降低刷新、断网、误关闭导致答案丢失的风险。
- 账号批量维护：批量重置密码、按教师导出学生、按班级导出名单。
- Word 导入导出任务化：大量试卷导出时改为后台任务，生成完成后下载。
- 数据归档：按学期归档旧班级、旧作业和旧答题记录，保持日常页面轻量。
- 审计增强：记录试卷修改前后的摘要、发布作业目标班级、批量导入结果。
- 数据库索引整理：围绕 `userId`、`classId`、`paperId`、`createdAt` 建立查询索引。

## P3 体验优化

- 教师工作台：登录后展示自己的班级、最近作业、待查看学情和快捷发布入口。
- 学生端移动适配：优化手机和平板答题体验。
- 考试模式：增加限时、自动交卷、禁止反复提交、提交后查看策略等配置。
- 错题与知识点：按题型、等级、知识点统计学生薄弱项。

## 生产环境建议配置

```bash
SESSION_MAX_AGE_DAYS=7
MAX_ATTEMPTS=100000
MAX_AUDIT_LOGS=10000
BACKUP_RETENTION=14
BACKUP_INTERVAL_HOURS=24
LOGIN_MAX_FAILURES=8
LOGIN_WINDOW_MINUTES=15
```

手动备份命令：

```bash
cd /opt/csppractice
npm run backup
```

管理员后台一键备份入口：管理后台 -> 系统设置 -> 数据备份。

后台恢复入口：管理后台 -> 系统设置 -> 数据备份 -> 查看备份 -> 恢复。后台恢复前会自动生成一份 `pre-restore` 安全备份。

命令行恢复时优先使用同一备份目录中的 `runtime.sqlite` 和 `papers.json`。恢复前先停止 PM2，并保留当前 `data/` 目录的一份副本。
